# Part 3 深入深度学习写作规范

> 本文件与 `writing-style.md` 配合使用，**不重复通用规则，只写 Part 3 特有的要求**。

---

## 核心定位

读者已经过了"入门"阶段，想真正看懂论文，甚至复现论文。这一部分**以论文为主线**：先交代论文在解决什么问题，再推导它的核心方法，最后讲清楚为什么这样设计。

代码是辅助理解的工具，不是目标。数学推导要完整，不能跳步。

---

## 章节结构

```
!!! info "参考资料"          ← 主要论文 + 关键引用，必须写

## 直觉 (Intuition)          ← 这篇论文在解决什么问题，一句话是什么贡献

## 问题背景                  ← 之前的方法有什么局限，这篇论文为什么出现

## 方法推导                  ← 核心数学，分步，每步说目的

## 关键设计决策              ← 为什么这样设计，论文的 ablation 说明了什么

## 代码（可选）              ← 核心模块的最简实现，不要写完整训练流程

## 局限与后续工作            ← 这个方法还有什么没有解决好
```

---

## 以论文为主线

每节必须从**"这篇论文在解决什么问题"**出发，而不是从定义出发。

**不好的开场：**
> VAE（Variational Autoencoder）是一种生成模型，它将编码器和解码器结合……

**好的开场：**
> 在 VAE 出现之前，自编码器可以重建输入，但没法生成新样本——隐空间是离散的、不连续的，随机采样一个点解码出来往往是噪声。VAE 的贡献是把隐空间变成一个有结构的概率分布，让采样变得有意义。

---

## 数学推导规范

推导必须完整，但每一步要说目的：

```markdown
**第一步**：引入变分下界，把难以计算的 $\log p(\mathbf{x})$ 转化为可优化的目标。

由于直接最大化 $\log p(\mathbf{x}) = \log \int p(\mathbf{x}|\mathbf{z}) p(\mathbf{z}) d\mathbf{z}$ 
对连续 $\mathbf{z}$ 不可处理，引入近似后验 $q_\phi(\mathbf{z}|\mathbf{x})$，通过 Jensen 不等式得到下界：

$$\log p(\mathbf{x}) \geq \mathbb{E}_{q_\phi}[\log p_\theta(\mathbf{x}|\mathbf{z})] - \text{KL}(q_\phi(\mathbf{z}|\mathbf{x}) \| p(\mathbf{z}))$$

**第二步**：解读这个下界的两项含义……
```

跳步是 Part 3 最常见的问题。哪怕推导是教科书级别的，也要写出来——读者在这里是第一次认真看推导。

---

## 符号对齐原则

**严格使用原论文符号**，不要自行重命名：

```markdown
!!! info "符号约定"
    本节符号遵循原论文 [Kingma & Welling, 2014]：
    - $\phi$：encoder 参数（论文中称为 recognition model）
    - $\theta$：decoder 参数（generative model）
    - $\mathbf{z}$：隐变量（latent variable）
```

如果引用多篇符号冲突的论文，在文章开头统一说明本节的约定。

## Ablation 的处理

论文里的消融实验是理解"为什么这样设计"的关键，要主动引用：

> 原论文 Table 3 的消融实验显示，去掉 LayerNorm 后训练直接发散——这说明在深层 Transformer 里，Post-Norm 的梯度问题比直觉上严重得多。

不要只介绍"这个模型有 A、B、C 三个组件"，要说"去掉 B 会怎样"。

## 代码的定位

Part 3 的代码是**理解辅助工具**，不是必选项：

- 代码只写核心算法（VAE 的重参数化、Diffusion 的加噪/去噪一步），不写完整训练循环
- 复杂的实现细节（如分布式训练）指向 Part 4 MLSys 章节
- 如果代码写出来反而让人更困惑，就不写

```python
# 重参数化技巧：让 z = μ + ε·σ 中的采样步骤可微
eps = torch.randn_like(sigma)
z = mu + eps * sigma  # 梯度通过 mu 和 sigma 流过，不通过随机采样
```

## "局限与后续工作"必写

Part 3 的读者是要做科研的，他们需要知道**这个方法还有什么问题**：

> VAE 生成的图像通常比较模糊。根本原因是优化的是像素级 MSE，而人眼对高频细节不敏感。这推动了后来 VQ-VAE 和扩散模型的出现——不再直接重建像素，而是在隐空间里建模。

这一节是连接"论文贡献"和"为什么要读下一篇论文"的桥梁。
